Port of the classic SimCity (based on SNES version) to MSX 1.

Screenshot of very beta version. Right bottom corner is a small debug info. Scanlines and gaussian blur set to mimic an old CRT TV.

After more than 25 years, I'm again programming a MSX! Suddenly I had an idea when reading something about the unreleased NES version of SimCity: what if I tried to squeeze this great game into the huge MSX 1 video restrictions? To make things even worse, what if I use MS-BASIC?

The MSX 1 hardware specs on a glance (screen 2):

  • 256 x 192 pixels;
  • only 16 colors (more on this later);
  • a fair amount of sprites: 256 (8x8) or 64 (16x16), 32 on screen simultaneously, but only 4 on the same line;
  • just one color per sprite.

As if 16 colors weren't enougth constraint, the palette choice is somewhat peculiar, for example there are two almost equal green shades, same as the two yellows, the only grey is very close to white... And things get worse: you cannot color each pixel individually, you have to take account of the neighborhood, each group of 8 pixels can only has two colors: foreground and background, if you try to add a third it replaces the color before (attribute clashing). This is a compromise necessary due to the tiny 16kb of memory (back in the 80's memory were very expansive).

Due to the grid nature of SimCity graphics (think of the first version, for SNES) I thought it could fit within these restrictions.

I'm using an emulator (openMSX) and coding on Visual Studio Code with Tandy Color Basic extension to syntax highlighting.
